# v0.2.30

## Release note

- Publishes the cleanarr runtime-hardening work from PR #7 as the first consumable release after the canceled `v0.2.29` tag-only attempt.

## Runtime hardening

- Added per-run Sonarr/Radarr catalog caching for tags, series, movies, and Sonarr episode lists to cut repeated ARR API calls during one cleanup pass.
- Tightened movie matching so token-subset fallbacks only apply when Plex and Radarr share external IDs, which prevents false-positive variant matches such as premiere or special-event titles being treated as the canonical movie.
- Reduced warning spam for watched Plex movies that sit outside known Radarr-managed paths; those are now skipped cleanly as unmanaged content instead of surfacing as noisy warnings.
- Added regression coverage for empty-body ARR delete success handling, movie-match safety, cache reuse, unmanaged-path skips, and Jellyfin webhook secret validation.
- Fixed the Jellyfin webhook import-path failure by defining and wiring `JELLYFIN_WEBHOOK_SECRET` and `JELLYFIN_WEBHOOK_SECRET_PREVIOUS`.

## Operational impact for AWS and home consumers

- AWS scheduled cleanup should see lower ARR call volume and cleaner unmatched-movie logs after this release.
- Home-repo Lambda deployments should consume `v0.2.30`, not the canceled `v0.2.29` tag.
- No manifest contract changes are required for existing AWS Lambda consumers, Plex webhook consumers, or home repo deployment workflows.
