# v0.2.29

## Runtime hardening

- Added per-run Sonarr/Radarr catalog caching for tags, series, movies, and Sonarr episode lists to cut repeated ARR API calls during one cleanup pass.
- Tightened movie matching so token-subset fallbacks only apply when Plex and Radarr share external IDs, which prevents false-positive variant matches such as premiere/special-event titles being treated as the canonical movie.
- Reduced warning spam for watched Plex movies that sit outside known Radarr-managed paths; those are now skipped cleanly as unmanaged content instead of surfacing as noisy warnings.
- Added regression coverage for empty-body ARR delete success handling, movie-match safety, cache reuse, unmanaged-path skips, and Jellyfin webhook secret validation.
- Fixed a webhook import-path failure by defining and wiring Jellyfin webhook secret configuration in `cleanarr/webhook_app.py`.
- Added robust Jellyfin webhook token gating to mirror Plex token behavior:
  - Supports `JELLYFIN_WEBHOOK_SECRET` and `JELLYFIN_WEBHOOK_SECRET_PREVIOUS`.
  - Rejects requests with mismatched/missing tokens with `401 Unauthorized`.
  - Allows both current and rotating previous secrets during staged secret rotation.
- Stabilized webhook route behavior with focused tests covering Jellyfin secret acceptance and rejection.
- Bumped package version to `0.2.29`.

## Operational impact for AWS and home consumers

- AWS scheduled cleanup should see lower ARR call volume and cleaner unmatched-movie logs after this release.
- Home-repo Lambda deployments can move to `v0.2.29` without manifest contract changes.
- No changes are required to AWS/ECS, Lambda, or home router/webhook ingress manifests for this release.
- Consumers should ensure webhook runtime environments include the new Jellyfin secret variable(s) where required.
- Runtime behavior is strictly more secure and safer by default for Jellyfin webhooks, with no runtime contract changes for existing Plex consumers or queueing modes.
